## Purpose

End-to-end skip bin hire management platform for Binned-IT Pty Ltd. Unifies daily operations (dispatch, bookings, fleet, drivers), customer/invoice management, financial intelligence, and compliance into a single multi-user web app — replacing Bin Manager + Xero exports + spreadsheets.

The product was originally a financial dashboard ("Binned-IT Dashboard Hub" v2.x). Per **PRD v5.0**, it has been re-anchored as **operations-first** with the dispatch board as the home screen and reporting as one tab among many. Current build version: **2.2.0**.

## Architecture pattern

**Single-page React SPA + serverless Edge Functions backed by Supabase Postgres.**

- All UI is client-rendered React. The Vite build outputs `dist/` which Vercel serves with a wildcard rewrite to `index.html`.
- Server-side responsibilities are split into 12 Vercel Edge Functions in `api/` (Anthropic proxy, Supabase Admin invites, Xero OAuth, scheduled crons for reminders/weekly digest/invoice-chase).
- Supabase Auth gates the app (`AuthGate` in `main.jsx`); RLS policies enforce role-based access. Public routes: `/book` (customer booking), `/driver/*` (driver portal handles its own auth).
- "Hardcoded fallback" pattern: every tab reads from `src/data/financials.js` when Supabase queries return empty/error — dashboard never blanks out.

Driver portal (`/drivers`) and `/investor` route are role-aware variants of the same SPA. `/book` is public.

## Top-level features

- **Dispatch board** (kanban with drag-drop) — operations home screen
- **Bookings** — customer-facing booking flow + ops side
- **Fleet management** — trucks, bins, maintenance log
- **Customer CRM** — accounts, churn risk, history
- **Invoices** — auto-generation, payment tracking, dunning
- **12-tab executive dashboard** — Snapshot, Revenue, Margins, Benchmarking, Competitors, Pricing, BDM, Fleet, Debtors, Cash Flow, Risk/EPA, Work Plan
- **AI assistant** (streaming Claude) — global ChatPanel + per-tab AIInsightsPanel
- **40+ alerts** generated by `analysisEngine.js`, categorised by severity
- **PWA** — installable, offline shell, mobile-responsive
- **PDF export** of dashboard tabs (`@media print` CSS)
- **Xero integration** (auth + invoice + payment sync — endpoints exist in `api/xero-*.js`)
- **Scheduled crons** — reminders (`0 8 * * *`), weekly digest (`0 20 * * 0`), invoice chase (`0 9 * * *`)
