## 8 — Quick reference for the AI

If you (a future AI agent) are picking up this codebase, your first reads should be:
1. `project-context.md` — distilled "do/don't" rules for working in SkipSync
2. `docs/index.md` — master navigation
3. `CLAUDE.md` — autonomous-action allowlist + conventions

Critical guardrails (also enforced in `project-context.md`):
- **No TypeScript.** Pure JS/JSX.
- **No CSS files / Tailwind / styled-components.** Inline `style={{}}` with `B.*` tokens from `src/theme.js` only.
- **`VITE_` vs no-`VITE_`.** Browser env vars must be prefixed `VITE_`. Server-only secrets must NOT be (`ANTHROPIC_API_KEY`, `SUPABASE_SERVICE_ROLE_KEY`, `VAPID_PRIVATE_KEY`).
- **Hardcoded fallback is sacred.** Every tab must render even when Supabase is unreachable — fall back to `src/data/financials.js`.
- **Migrations are idempotent + uniquely prefixed.** See `supabase/migrations/README.md`.
- **`master` is the deploy branch.** `git push origin master` ships to production. Always run `npm run build && npm test` first.
