## 7 — Sync on another PC

Order matters. Run from a fresh clone unless noted otherwise.

### 7.1 — Clone + install
```bash
git clone https://github.com/markgbeddoe-dotcom/BinnedIT-Hub.git
cd BinnedIT-Hub
npm install
```

### 7.2 — Pull Vercel env vars into `.env.local`
You need the Vercel CLI logged in as the project owner (or a team member). The CLI prompts a browser-based device-code login the first time.

```bash
npx vercel login                       # if not already authenticated
npx vercel link --yes --project binnedit-hub
npx vercel env pull .env.local         # downloads ALL env vars for the linked project
```

This populates `.env.local` with `VITE_SUPABASE_URL`, `VITE_VAPID_PUBLIC_KEY`, `ANTHROPIC_API_KEY`, `VAPID_PRIVATE_KEY`, etc. The file is git-ignored.

### 7.3 — Install Playwright browsers (one-time per PC)
```bash
npm run test:e2e:install
```
Downloads ~300–500 MB to `%LOCALAPPDATA%\ms-playwright\` (Windows) or `~/.cache/ms-playwright/` (macOS/Linux).

### 7.4 — Verify the setup
```bash
npm run build           # 0 errors expected
npm test                # 7 unit tests passing
npm run test:e2e        # 2 e2e tests passing (auto-spawns dev server)
```

## 8 — Quick reference for the AI

If you (a future AI agent) are picking up this codebase, your first reads should be:
1. `project-context.md` — distilled "do/don't" rules for working in SkipSync
2. `docs/index.md` — master navigation
3. `CLAUDE.md` — autonomous-action allowlist + conventions

Critical guardrails (also enforced in `project-context.md`):
- **No TypeScript.** Pure JS/JSX.
- **No CSS files / Tailwind / styled-components.** Inline `style={{}}` with `B.*` tokens from `src/theme.js` only.
- **`VITE_` vs no-`VITE_`.** Browser env vars must be prefixed `VITE_`. Server-only secrets must NOT be (`ANTHROPIC_API_KEY`, `SUPABASE_SERVICE_ROLE_KEY`, `VAPID_PRIVATE_KEY`).
- **Hardcoded fallback is sacred.** Every tab must render even when Supabase is unreachable — fall back to `src/data/financials.js`.
- **Migrations are idempotent + uniquely prefixed.** See `supabase/migrations/README.md`.
- **`master` is the deploy branch.** `git push origin master` ships to production. Always run `npm run build && npm test` first.
